	Программа Scrambler предназначена для шифрования и последующей расшифровки небольших (до 50 Мбайт) файлов. Для её работы необходимы два файла: "Scrambler.jar" -- исполняемый java-архив (для запуска на Вашем устройстве должна быть установлена виртуальная java машина) и "keys.properties" -- файл с настройками.
	Вы можете свободно использовать как готовую программу, так и исходные коды (частями или полностью) по своему усмотрению. Автор не несёт ответственности за последствия применения программы.
	
	Для того, чтобы выполнить шифрование, необходимо нажать кнопку "Encrypt" и выбрать файл в диалоговом окне. При этом с файлом выполняются в определённой последовательности несколько раз следующие действия:
		1. Создаётся генератор случайных чисел с определённой точкой вхождения (создаёт одинаковые последовательности псевдослучайных чисел при каждом запуске; для точек вхождения используются значения key.0 .. key.4 из файла настроек). Создаётся новый пустой массив (на самом деле ArrayList, а не обычный массив) и из файла "случайные" байты переносятся по очереди в конец нового массива. Если размер файла больше чем определённое число байт (значение maxpiece из файла настроек), то сначала файл делится пополам, затем каждая часть снова делится пополам, пока размер каждой части не станет меньше maxpiece, а затем с каждой частью проводятся вышеописанные операции по отдельности.
		2. В "случайные" места файла вставляются случайные байты. Их количество определяется параметрами inclusions.0 и inclusions.1 из файла настроек.
		3. Файл сжимается в zip-архив.
	По окончании этих операций в конец файла добавляется определённая короткая последовательность байт ("подпись") и изначально открытый файл перезаписывается результирующим зашифрованным.
	Для расшифровки файла следует нажать кнопку "Decrypt" и выбрать зашифрованный файл в диалоговом окне. При этом с файлом выполняются все действия, обратные тем, которые выполняются при шифровании в обратной последовательности. Стоит отметить, что поскольку при шифровании в файл вставляются байты в случайные места, то при расшифровке байты из предполагаемых мест вставки будут удаляться. При попытке расшифровать этой программой незашифрованный файл он может быть необратимо повреждён. Для предотвращения этого проверяется наличие "подписи" в конце файла, если её нет, расшифровка не начнётся. Но если Ваш файл случайно оканчивается такой же последовательностью байт как в "подписи" (или он был ранее зашифрован данной программой с другими настройками, "подпись" всегда одинакова) и Вы попытаетесь его расшифровать, то если этот файл был нужен... примите мои искренние соболезнования.
	
	Истинная цель создания этой программы, конечно, не шифрование файлов. Изначально я написал первую версию этой программы для закрепления навыков работы с коллекциями, вводом-выводом и многопоточностью. Сейчас, когда я довёл эту программу до ума, в ней используются следующие технологии из Java Core (и не только):
		1. Для создания пользовательского интерфейса использовался JavaFX API.
		2. Для обработки и промежуточного хранения данных используется ArrayList. Применяется как доступ к элементам по индексу, так и перебор (прямой и обратный) с помощью итераторов.
		3. Для доступа к файлам используются различные классы из пакета java.io.
		4. При перемешивании байтов задача выполняется в параллельных потоках для ускорения процесса. Используется ForkJoin API. (В старой версии программы я использовал CountDownLatch и разбиение на фиксированное число подзадач.)
		5. Написан unit-тест для проверки корректности работы отдельных методов шифровки/расшифровки (3 теста, по одному для каждой пары методов) с использованием библиотеки JUnit 4.
		6. Поскольку методы из предыдущего пункта объявлены в классе "scrambler.core.Core" с модификатором доступа private, доступ к ним осуществлялся при помощи Java Reflection API.
